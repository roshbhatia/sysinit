## ADDED Requirements

### Requirement: Codex profiles set reasoning_effort per profile
`~/.codex/config.toml` MUST contain at least two named profiles: `default` (with `reasoning_effort = "low"`) and `spec` (with `reasoning_effort = "high"` and `model_reasoning_summary = "detailed"`). The profile set SHALL be generated by `codex.nix` via the `programs.codex.settings.profiles` attrset (or whatever the home-manager module exposes for profiles).

#### Scenario: Profiles materialize
- **WHEN** `nh darwin switch .` completes
- **THEN** `cat ~/.codex/config.toml` shows `[profiles.default]` and `[profiles.spec]` sections with the declared reasoning settings

#### Scenario: Default profile selected
- **WHEN** `codex` is launched without `--profile`
- **THEN** the active profile is `default` and `reasoning_effort` is `low`

#### Scenario: Spec profile selected
- **WHEN** `codex --profile spec` is launched (or the equivalent profile-selector in Codex CLI)
- **THEN** the active profile is `spec`, `reasoning_effort` is `high`, and reasoning summaries are visible in the session UI

### Requirement: Profile settings are inert if Codex CLI does not support them
If a future Codex CLI release removes or renames `reasoning_effort` or `model_reasoning_summary`, the home-manager-rendered TOML SHALL still parse without error (unknown keys are tolerated by TOML parsers, and Codex CLI's behavior on unknown settings is documented to be silent ignore per the audit). The settings continue to live in `codex.nix` until the audit re-runs.

#### Scenario: Unknown setting tolerated
- **WHEN** Codex CLI receives a config containing a setting it doesn't recognize
- **THEN** Codex starts without error (this is a Codex CLI contract, not a sysinit requirement — but sysinit relies on it)
