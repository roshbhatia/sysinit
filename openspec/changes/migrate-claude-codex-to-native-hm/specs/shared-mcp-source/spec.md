## ADDED Requirements

### Requirement: programs.mcp.servers is the canonical MCP source for claude-code and codex
The system SHALL define MCP server entries in `programs.mcp.servers` and enable both `programs.claude-code.enableMcpIntegration` and `programs.codex.enableMcpIntegration` to consume them.

#### Scenario: MCP servers defined once
- **WHEN** a new MCP server is added to `programs.mcp.servers`
- **THEN** it appears in both the Claude Code and Codex configurations without additional changes

#### Scenario: Other tools unaffected
- **WHEN** `programs.mcp.servers` is updated
- **THEN** gemini, amp, cursor, goose, crush, opencode, copilot-cli configurations are not changed (they still use `lib/mcp.nix` formatters)

### Requirement: lib/mcp.nix formatters remain for non-native tools
The system SHALL preserve `lib/mcp.nix` and `mcp.nix` to continue serving the 7 LLM tools that do not have native HM MCP integration.

#### Scenario: Gemini config still receives MCP servers
- **WHEN** home-manager builds the configuration
- **THEN** `~/.config/gemini/settings.toml` contains the MCP server section generated by `formatForGemini`

#### Scenario: No duplication between programs.mcp and mcp.nix
- **WHEN** the configuration is built
- **THEN** MCP server definitions exist in exactly one place: `programs.mcp.servers` (which feeds claude-code and codex) while `mcp.nix` reads or mirrors from it for the remaining tools, OR `mcp.nix` continues to define the servers and `programs.mcp.servers` is populated from that same source
