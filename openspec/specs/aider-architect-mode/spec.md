## Requirements
### Requirement: Architect and editor models are configured separately
`.aider.conf.yml` (generated by `aider.nix` via the home-manager `programs.aider-chat.settings`) MUST set `architect-model` to a stronger Anthropic model (default: `anthropic/claude-sonnet-4-5`) and `editor-model` to a cheaper Anthropic or OpenAI model (default: `anthropic/claude-haiku-4-5`). The split SHALL be enabled by default (`architect: true`).

#### Scenario: Architect/editor split active
- **WHEN** `aider --help` is invoked after activation
- **THEN** the help output reflects the configured architect and editor models

#### Scenario: Both models reachable
- **WHEN** `aider --architect --message "echo test"` is run interactively
- **THEN** aider proceeds without an "architect-model unreachable" error (assuming valid Anthropic credentials are configured)

### Requirement: CONVENTIONS.md is sourced from the repository's AGENTS.md
The aider working directory's `CONVENTIONS.md` MUST be a symlink or generated file that mirrors the repository's root `AGENTS.md`. Aider sessions started under sysinit pick up the same convention surface every other harness reads. The symlink/file SHALL be created by `aider.nix` via `home.file.".aider/CONVENTIONS.md"` (or equivalent) and SHALL NOT shadow per-project `CONVENTIONS.md` files when present.

#### Scenario: Default conventions present
- **WHEN** aider is launched in a directory with no per-project `CONVENTIONS.md`
- **THEN** the global `~/.aider/CONVENTIONS.md` (or whatever path the home-manager module writes) provides the canonical convention source matching `AGENTS.md`

#### Scenario: Per-project override
- **WHEN** aider is launched in a directory that has its own `CONVENTIONS.md`
- **THEN** that local file takes precedence and the global symlink is ignored
